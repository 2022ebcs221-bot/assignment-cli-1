4. File Descriptor Observation

Command used:

echo "Writing using file descriptor 3" >&3

Output observed:

Observation:
File descriptor 3 was opened and used to write data into custom_fd.log. This shows that Linux supports custom file descriptors in addition to standard input, output, and error.

9. Explanation of Linux File I/O

Linux manages file I/O using file descriptors. Standard input is file descriptor 0, standard output is file descriptor 1, and standard error is file descriptor 2. Additional file descriptors such as 3 can be opened for custom file operations.

Output redirection sends normal command output to a file, while error redirection sends error messages to a separate file. This separation is useful for debugging application logging and file access issues.
